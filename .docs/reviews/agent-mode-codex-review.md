# Agent Mode Review

## 1. Issues Found

- **critical**: Claude project directory decoding in `generateClaudeProjectRules()` is lossy and can misclassify active projects as “orphan” sessions. The decoder at [electron/main/rules-engine.ts:156](/Users/nowcoder/Documents/MyCode/services/MagicBroom/electron/main/rules-engine.ts:156) assumes every `-` in `~/.claude/projects/*` is a path separator. Real Claude slugs are not reversible that way: they collapse path separators and real characters like `-`, `_`, and `.` into the same slug. I spot-checked a real session log whose `cwd` is `/Users/nowcoder/Documents/Nowcoder/nc_hybrid/nc-niuzhao/fe-nowfind/.claude/worktrees/festive-raman`, but its project directory is named `-Users-nowcoder-Documents-Nowcoder-nc-hybrid-nc-niuzhao-fe-nowfind--claude-worktrees-festive-raman`. The current decoder turns that into `/Users/nowcoder/Documents/Nowcoder/nc/hybrid/nc/niuzhao/fe/nowfind//claude/worktrees/festive/raman`, which can never match the real project. On this machine, only 3 of 36 Claude project dirs decoded to an existing path; 33 would be treated as orphan. Because the orphan rule is emitted as `risk: safe` at [electron/main/rules-engine.ts:185](/Users/nowcoder/Documents/MyCode/services/MagicBroom/electron/main/rules-engine.ts:185), [electron/main/rules-engine.ts:189](/Users/nowcoder/Documents/MyCode/services/MagicBroom/electron/main/rules-engine.ts:189), users can be told active session histories are safe to delete.

- **warning**: Several “cache” rules point at whole tool home directories instead of cache-only subpaths, so they can delete important user data. The strongest examples are [rules/agent/ai-tools.json:15](/Users/nowcoder/Documents/MyCode/services/MagicBroom/rules/agent/ai-tools.json:15) (`~/.cursor`) and [rules/agent/ai-tools.json:25](/Users/nowcoder/Documents/MyCode/services/MagicBroom/rules/agent/ai-tools.json:25) (`~/.codex`). On this machine, `~/.cursor` contains user-authored plan files, and `~/.codex` contains auth, prompts, rules, plugins, sessions, and memories in addition to cache. Calling these “缓存” and marking `~/.codex` as `safe` is misleading. The same structural risk likely applies to whole-dir rules such as [rules/agent/ai-tools.json:45](/Users/nowcoder/Documents/MyCode/services/MagicBroom/rules/agent/ai-tools.json:45), [rules/agent/ai-tools.json:55](/Users/nowcoder/Documents/MyCode/services/MagicBroom/rules/agent/ai-tools.json:55), [rules/agent/ai-tools.json:65](/Users/nowcoder/Documents/MyCode/services/MagicBroom/rules/agent/ai-tools.json:65), and [rules/agent/ai-tools.json:95](/Users/nowcoder/Documents/MyCode/services/MagicBroom/rules/agent/ai-tools.json:95).

- **warning**: The grouped UI can keep stale selections across rescans. `selectedItems` is local component state at [src/app/pages/AgentMode.tsx:24](/Users/nowcoder/Documents/MyCode/services/MagicBroom/src/app/pages/AgentMode.tsx:24), but `handleScan()` at [src/app/pages/AgentMode.tsx:47](/Users/nowcoder/Documents/MyCode/services/MagicBroom/src/app/pages/AgentMode.tsx:47) does not clear it. The action bar then displays `selectedItems.size` at [src/app/pages/AgentMode.tsx:138](/Users/nowcoder/Documents/MyCode/services/MagicBroom/src/app/pages/AgentMode.tsx:138) and uses that raw size to enable the CTA at [src/app/pages/AgentMode.tsx:157](/Users/nowcoder/Documents/MyCode/services/MagicBroom/src/app/pages/AgentMode.tsx:157). After a rescan, the UI can show a non-zero selected count for IDs that are no longer in `agentItems`, and the clean button can remain enabled even when nothing in the current result set is actually selected.

- **info**: The orphan/stale grouping itself is understandable, but the surrounding copy is less clear than the grouping. The groups are named well at [src/app/pages/AgentMode.tsx:41](/Users/nowcoder/Documents/MyCode/services/MagicBroom/src/app/pages/AgentMode.tsx:41), [src/app/pages/AgentMode.tsx:42](/Users/nowcoder/Documents/MyCode/services/MagicBroom/src/app/pages/AgentMode.tsx:42), and [src/app/pages/AgentMode.tsx:43](/Users/nowcoder/Documents/MyCode/services/MagicBroom/src/app/pages/AgentMode.tsx:43). The weaker part is that the summary says “已识别工具” at [src/app/pages/AgentMode.tsx:118](/Users/nowcoder/Documents/MyCode/services/MagicBroom/src/app/pages/AgentMode.tsx:118) and the CTA says “清理 Agent 缓存” at [src/app/pages/AgentMode.tsx:162](/Users/nowcoder/Documents/MyCode/services/MagicBroom/src/app/pages/AgentMode.tsx:162), even though the first two groups are Claude session histories and the third group also includes full model stores. - **info**: The `~/.claude/projects/` scan is efficient enough at current scale. The implementation is linear and somewhat syscall-heavy because it uses `readdir()` plus `stat()` on each entry at [electron/main/rules-engine.ts:135](/Users/nowcoder/Documents/MyCode/services/MagicBroom/electron/main/rules-engine.ts:135), [electron/main/rules-engine.ts:149](/Users/nowcoder/Documents/MyCode/services/MagicBroom/electron/main/rules-engine.ts:149), [electron/main/rules-engine.ts:161](/Users/nowcoder/Documents/MyCode/services/MagicBroom/electron/main/rules-engine.ts:161), and [electron/main/rules-engine.ts:174](/Users/nowcoder/Documents/MyCode/services/MagicBroom/electron/main/rules-engine.ts:174). But on this machine, scanning 36 project dirs and 255 `.jsonl` files with the same access pattern finished in about 89 ms. I would not treat this as a blocking performance problem unless users routinely accumulate thousands of project dirs.

## 2. Missing AI Tools That Should Be Added

- **Tabnine**: This is the clearest omission because the settings UI already treats `~/.tabnine` as a known AI tool preset at [src/app/pages/Settings.tsx:306](/Users/nowcoder/Documents/MyCode/services/MagicBroom/src/app/pages/Settings.tsx:306), but `rules/agent/ai-tools.json` has no Tabnine rule. Official Tabnine CLI docs place settings under `~/.tabnine/agent/settings.json`, so the product is clearly using its own home directory. Add it, but prefer scoped cache/model/history rules instead of a blanket “safe cache” rule. Source: [Tabnine CLI settings](https://docs.tabnine.com/main/getting-started/tabnine-cli/features/settings)

- **Gemini CLI**: Official Gemini CLI docs store MCP/server config in `~/.gemini/settings.json` and support checkpointed sessions, which makes `~/.gemini` a meaningful Agent-mode target. Add it, but split config/history/cache if possible instead of deleting the whole directory. Source: [Gemini CLI](https://github.com/google-gemini/gemini-cli)

- **Cline**: Official Cline CLI docs use `~/.cline/data/` for local state, and the editor extension also keeps data under host-editor global storage. Cline is mainstream enough now that Agent mode should recognize it explicitly. Source: [Cline CLI configuration](https://docs.cline.bot/cline-cli/configuration)

- **Roo Code**: Roo Code is another major agent workflow that stores state under host-editor global storage. The official repo discusses its storage under `rooveterinaryinc.roo-cline`, which is a good signal that Agent mode should recognize Roo-specific data separately from generic editor data. Source: [Roo Code storage discussion](https://github.com/RooCodeInc/Roo-Code/issues/1578)

- **Sourcegraph Cody / Amp**: Sourcegraph documents local code indexes under `.../globalStorage/sourcegraph.cody-ai/symf/indexroot/` on macOS. That is a good safe-cache target and also covers the current Cody/Amp product line that many users still know by the older Cody name. Source: [Sourcegraph local indexing](https://sourcegraph.com/docs/cody/capabilities/local-indexing)

## 3. Suggested Improvements

- Stop trying to reconstruct the original Claude project path from the directory slug. Use authoritative metadata from the newest `.jsonl` inside each project dir instead; the session entries already contain `cwd`. If that metadata cannot be read, fall back to an “unknown session” classification instead of “orphan-safe”.

- Add tests for Claude project names containing real hyphens, underscores, dot-prefixed directories, and worktree paths. The current tests only cover wildcard expansion, so the most fragile Agent-mode logic has no regression coverage.

- Split broad home-directory rules into smaller buckets. For example, separate `~/.codex/cache` from sessions/prompts/plugins/auth, and separate Cursor cache from user-authored plans/settings. Any rule that still includes auth, config, or conversation history should be `warning` or `danger`, not `safe`.

- Clear local selection state on every rescan, or compute the selected count and button enabled-state from `agentItems.filter((i) => selectedItems.has(i.id))` rather than from `selectedItems.size`.

- Tighten the wording around the groups: rename the summary metric from “已识别工具” to something like “可清理项”, rename the CTA from “清理 Agent 缓存” to “清理选中项”, and add one-line helper text that explains “孤儿会话” vs “陈旧会话”. Sorting stale items by “days inactive” or by size would also make the grouping easier to scan.

- Minor only: if you want to trim scan overhead later, switch `readdir(projectsDir)` to `readdir(projectsDir, { withFileTypes: true })` and avoid one `stat()` per top-level entry. I would treat this as cleanup, not as a fix.

- User-facing naming should catch up with current products: relabel “Codeium” as “Codeium / Windsurf” rather than duplicating it as a separate rule. Source: [Windsurf docs](https://docs.windsurf.com/command/codeium-overview)
